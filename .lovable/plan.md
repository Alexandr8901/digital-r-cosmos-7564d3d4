# Итерация 3 — Кабинеты Самозанятого, ИП и ООО

Объём огромный (≈80+ страниц). Чтобы не превратить это в шесть часов генерации мусора, разобью на 3 блока и сделаю их последовательно, по подтверждению. Архитектура и дизайн-токены — без изменений, продолжаю citizen-стиль (sidebar + topbar + kit-компоненты + recharts + framer-motion + mock-данные через seedrandom).

## Общая инфраструктура (делаю в любом случае первой)

- **Переключатель режимов** (Гражданин / Самозанятый / ИП / ООО) в topbar. Хранит активный режим в `user_settings` (key=`active_workspace`). Мгновенное переключение — `router.navigate` на корень режима, без re-auth. Доступные режимы вычисляются по `user_roles` (роли: `citizen`, `self_employed`, `ip`, `ooo`). Миграция: добавить новые значения в `app_role` enum + расширить `handle_new_user` не трогать (роли добавляются вручную через UI «активировать режим»).
- **Глобальный поиск** (`⌘K` / `Ctrl+K`) — `cmdk`-диалог, ищет по mock-индексу: клиенты, товары, платежи, документы, договоры, организации, сотрудники, операции.
- **kit-компоненты v2**: `DataTable` (сортировка, фильтры, экспорт CSV/Excel/PDF), `Money`, `StatusBadge`, `FilterBar`, `DateRangePicker`, `Sparkline`, `DashboardGrid` (drag&drop виджетов через `@dnd-kit`), `IntegrationStatus` (отображение «через официальный API» / «в разработке»), `AuditTrail`.
- **Расширение mock-слоя**: `src/lib/mock/{selfemployed,ip,ooo}.ts` — детерминированные данные на seedrandom.

## Блок A — Кабинет Самозанятого `/_authenticated/self-employed/*`

12 страниц: `index` (Dashboard), `income`, `clients`, `services`, `invoices`, `invoices.new` (мастер), `documents`, `payments`, `qr`, `analytics`, `ai`, `settings`, `support`.

Особое внимание:
- Dashboard: KPI-карточки (доход общий/месяц/год, клиенты, счета) + последние оплаты + AI-рекомендации + быстрые действия.
- Мастер выставления счёта — 6 шагов с framer-motion (как register wizard): клиент → услуги → стоимость+комментарий → срок+способ → QR+предпросмотр → отправить.
- Аналитика: recharts (line/bar/pie), сравнение периодов.

## Технические решения

- Все режимы используют общий `WorkspaceShell` (sidebar+topbar) с разными `navItems`. Активная организация/режим хранится в Context + `user_settings`.
- Все интеграционные точки (ЦБ РФ, ФНС, СБП, банки) — единый компонент `<IntegrationStatus status="pending-official-api" />` с текстом «Подключается через официальное API после соглашения».
- Drag & Drop — `@dnd-kit/core` + `@dnd-kit/sortable`.
- Экспорт PDF — `jspdf` + `jspdf-autotable`. Excel — `xlsx`. CSV — самописный.
- Глобальный поиск — `cmdk` (уже в shadcn).
- Темы, skeleton, empty/error/success — переиспользуем существующие kit-компоненты.
- Автосохранение форм — `react-hook-form` + debounced save в `localStorage` по ключу маршрута.

## Миграции БД

1. Расширить `app_role` enum: добавить `self_employed`, `ip`, `ooo`.
2. Таблица `organizations` (для ИП/ООО): name, inn, kpp, ogrn, address, type, owner_id + RLS.

## Что НЕ делаю (вне scope)

- Мобильное приложение.
- Кабинет администратора платформы (отдельная итерация).
- Marketplace как магазин (только landing-страница раздела с заглушкой).
