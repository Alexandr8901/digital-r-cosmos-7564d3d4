# Iteration 4 — Финансовая инфраструктура, Бизнес-экосистема, Developer Platform, Marketplace

Огромный объем (4 крупных блока, ~120 новых маршрутов). Перед стартом подтверди объем — иначе разобью на под-итерации.

## Принципы (не меняются)

- Та же дизайн-система (`workspace/kit`, `citizen/kit`), темы, токены.
- Все «внешние» подключения — через `IntegrationStatus` (Не подключено / Доступно через официального партнера / Подключено). Никакой имитации реальных платежей.
- Mock-данные детерминированные (seedrandom), как в `lib/mock/business.ts`.
- Skeleton / Empty / Error / Success состояния в каждом разделе через общий `kit`.

## Блок A — Финансы (главный раздел `/citizen/finance/*`, ~22 страницы)

Расширяет уже существующий citizen-кабинет, добавляя единый финансовый хаб.

- `route.tsx` (sidebar c подразделами) + `index.tsx` (Dashboard: общий баланс, ЦР, банки, доходы/расходы, операции, переводы, автоплатежи, документы, AI quick actions)
- Подразделы: `payments`, `transfers`, `digital-ruble/{index,balance,history,transfers,receive,qr,settings,limits,banks,security,integrations}`, `qr`, `sbp`, `bank-accounts`, `history`, `templates`, `autopayments`, `recipients`, `requisites`, `documents` (внутри финансов — счета/договоры/акты/УПД/архив/подписи), `export`, `ai`, `calendar`, `security`, `notifications`
- Мастер переводов (5 шагов, framer-motion)
- Мастер создания документа (тип → контрагент → товары → налоги → предпросмотр)
- Календарь (`@fullcalendar/*` или собственная сетка — используем собственную, без новой зависимости)

## Блок B — Бизнес (новый главный раздел `/business/*`, ~30 страниц)

Это отдельная workspace-зона поверх существующих ip/ooo. Объединяет общие бизнес-функции.

- Dashboard с настраиваемыми виджетами (`@dnd-kit` уже установлен)
- CRM: leads, contacts, companies, deals, pipeline (Kanban), tasks, notes, files, tags, custom-fields
- Каталог товаров и услуг (CRUD + поиск + импорт CSV)
- Склад, закупки, поставщики
- Команда, проекты, задачи, календарь
- Финансы бизнеса, бухгалтерия (архитектура под интеграции)
- BI Аналитика (recharts: дашборды, сводные, тепловые карты, прогнозы)
- AI Центр (чат + шаблоны запросов)
- AI Автоматизация (визуальный конструктор сценариев — react-flow)
- Marketplace (сразу здесь — карточки приложений)
- Уведомления (центр)

## Блок C — Developer Platform (`/developers/*`, ~25 страниц)

- Dashboard разработчика (метрики, запросы, ключи, webhooks)
- API Center: REST, GraphQL, Explorer, OpenAPI, Postman, CLI docs
- SDK страница (12 языков с табами и установкой)
- API Keys (CRUD, ротация, IP allow list)
- OAuth (приложения, scopes, consent screen mock)
- Webhooks (создание, события, история, тест, подпись)
- Events каталог (платеж/документ/счет/...)
- Sandbox (тестовые сущности и генератор данных)
- Documentation портал (поиск, версии, навигация)
- Status page (uptime, инциденты)
- Logs, Rate limits, Versioning, Marketplace для расширений, Monitoring
- Безопасность (JWT/Passkeys/Signature/Audit)

## Блок D — Marketplace (`/marketplace/*`, ~20 страниц)

Публичный + кабинет:

- Главная Marketplace, Каталог по категориям, страница приложения (с галереей, отзывами, разрешениями)
- Подключение приложения (мастер)
- Интеграции: подключенные/доступные/рекомендуемые, конструктор интеграций (визуальный)
- Конструктор автоматизации (Workflow Builder — react-flow)
- Шаблоны процессов
- AI Marketplace
- Партнерская программа (кабинет)
- Кабинет разработчика приложений (мои приложения, публикация, модерация, отзывы, финансы)
- Подписки, центр обновлений, центр безопасности
- Каталог официальных интеграций
- Глобальный поиск

## Технические детали

- Новые зависимости: `reactflow` (для визуальных конструкторов), `prismjs` + `react-syntax-highlighter` (подсветка кода в Dev Platform).
- Новые компоненты в `src/components/`: `finance/kit.tsx`, `developer/kit.tsx` (CodeBlock, EndpointCard, KeyRow), `marketplace/kit.tsx` (AppCard, CategoryGrid, ReviewItem), `automation/flow-canvas.tsx`.
- Mock layer: `src/lib/mock/finance.ts`, `lib/mock/developer.ts`, `lib/mock/marketplace.ts`.
- Глобальный поиск (cmdk) расширяется индексами из новых модулей.
- Все workspace-shell-ы переиспользуются; добавляются записи в `app.tsx` навигации.

## Что НЕ делается (явно)

- Никаких реальных Stripe/банковских/ФНС/ЦБ интеграций.
- Никаких новых БД-таблиц для бизнес-сущностей — всё mock до момента официального API.
- Никаких реальных OAuth-приложений / webhook-приёмников — только UI и архитектура.

## Объем и риски

- ~120 новых файлов маршрутов + ~10 компонентов/моков + 2 новые зависимости.
- Один проход = очень крупный коммит. Рекомендую разбить на 2 итерации:
  - **4A**: Блок A (Финансы) + Блок B (Бизнес) — финтех-расширение.
  - **4B**: Блок C (Developer Platform) + Блок D (Marketplace) — экосистема.
